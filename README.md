## Описание проекта
Проект представляет собой клиент-серверное приложение для сокращения длинных URL-адресов. Серверная часть реализует полноценный REST API, а клиентская предоставляет удобный консольный интерфейс (CLI) для пользователя. 

Проект полностью написан на языке **Clojure**, что демонстрирует навыки применения функциональной парадигмы программирования, работы с неизменяемыми структурами данных и функционального подхода к маршрутизации веб-запросов.

Основные технологии, применённые в проекте:
- **Clojure** — основной язык разработки (Lisp-диалект для JVM).
- **Ring & Compojure** — стандартный стек для создания HTTP-сервера и декларативной маршрутизации в Clojure.
- **clj-http** — функциональная обёртка для выполнения HTTP-запросов на стороне клиента.
- **SQLite & clojure.java.jdbc** — встроенная реляционная база данных для персистентного хранения ссылок.
- **Leiningen** — система автоматизации сборки и управления зависимостями.

## Задание
В рамках работы было необходимо спроектировать службу сокращения ссылок, состоящую из независимых клиента и сервера:

1. **Разработать REST API сервер**, обрабатывающий следующие запросы:
   - `POST /` — принимает длинный URL, генерирует короткий хэш, сохраняет в БД и возвращает короткий URL.
   - `GET /:short-url` — возвращает длинный URL по короткому (или ошибку, если не найдено).
   - `PUT /:short-url` — изменяет длинный URL для существующей короткой ссылки.
   - `DELETE /:short-url` — удаляет связку из базы данных.
2. **Разработать CLI-клиента** (консольное приложение) с интерактивным меню:
   - Доступные действия: *Создать, Показать, Изменить, Удалить, Выйти*.
   - Клиент должен принимать ввод от пользователя, отправлять HTTP-запросы на сервер и форматировано выводить ответы (например, `Ответ: OK` или `Ответ: Ошибка`).

## Результат выполнения и запуск

Проект успешно собирается в единый исполняемый `.jar` файл, который может работать как в режиме сервера (при передаче порта), так и в режиме клиента. При первом запуске сервера база данных `url_shortener.db` инициализируется автоматически.
